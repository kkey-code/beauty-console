# 门店后台管理系统项目介绍

## 1. 项目概述

本项目是一个面向门店经营场景的后台管理系统，当前重构项目目录为 `beauty-console`。系统主要用于管理门店用户、员工、客户、服务项目、预约、服务订单、支付记录、库存物品和库存流水。

项目采用前后端分离思路，后端提供 REST API，前端通过登录接口获取 JWT token，后续访问后台接口时携带 token 完成身份认证。

当前后端重点目标是：

- 拆分清晰的 Maven 多模块结构。
- 使用 DTO 接收请求参数，使用 VO 返回前端数据。
- 使用 Entity 对应数据库表。
- 使用 MyBatis-Plus 快速生成基础 CRUD 骨架。
- 在关键业务中补充校验、事务、金额计算、库存变更和登录认证。

## 4. 各模块职责

| 模块 | 职责 |
|---|---|
| `store-common` | 存放通用返回对象、分页结果、上下文、异常、JWT 工具、配置属性等公共能力 |
| `store-pojo` | 存放 DTO、VO、Entity、Enum，不直接写业务逻辑 |
| `store-server` | 存放 Controller、Service、Mapper、拦截器、配置类和启动类，是主要业务模块 |

## 5. 核心业务模块

| 模块 | 说明 |
|---|---|
| 用户管理 | 后台账号登录、分页查询、新增、修改、启停用，角色覆盖 admin、staff、readonly |
| 员工管理 | 维护门店员工资料，供账号、预约、订单服务人员关联 |
| 客户管理 | 维护客户姓名、电话、性别、生日、等级、来源等资料 |
| 服务项目 | 维护门店可售卖或可预约的服务项目 |
| 预约管理 | 创建预约、维护预约项目明细、查询预约列表和详情 |
| 服务订单 | 新建订单、从预约生成订单、维护订单明细、校验订单金额和支付状态 |
| 支付记录 | 新增收款、退款、作废记录，并同步反写订单已付金额、欠款金额、支付状态 |
| 库存物品 | 维护库存 SKU、当前库存、安全库存、成本价、供应商 |
| 库存流水 | 记录入库、出库、盘点、报损、退货，并同步更新库存数量 |

## 7. 接口约定

后台接口统一使用 `/admin` 前缀，登录接口除外也在 `/admin/users/login` 下。

统一返回格式：

```json
{
  "code": 1,
  "msg": null,
  "data": {}
}
```

分页返回格式：

```json
{
  "code": 1,
  "msg": null,
  "data": {
    "total": 100,
    "records": []
  }
}
```

登录后请求头：

```http
token: Bearer <登录返回的token>
```

或者在当前拦截器兼容逻辑下：

```http
token: <登录返回的token>
```

请求头名称由配置控制：

```yaml
sky:
  jwt:
    admin-token-name: token
```

## 8. 典型业务流程

### 8.1 登录流程

```text
前端提交 username/password
-> SysUserController
-> SysUserService
-> 查询 sys_user 和员工信息
-> 校验密码
-> 生成 JWT
-> 返回 LoginUserVO
```

### 8.2 预约转订单流程

```text
查询预约主表
-> 查询预约明细
-> 生成订单号
-> 复制必要客户、员工、服务时间信息
-> 根据预约明细生成订单明细
-> 汇总金额
-> 校验订单金额
-> 保存订单主表
-> 保存订单明细
```

### 8.3 支付记录流程

```text
新增支付记录
-> 查询订单
-> 校验订单是否存在、是否取消
-> 校验支付状态和金额
-> 保存 payment_record
-> 根据成功收款或退款反写订单金额
-> 刷新 payStatus 和 debtStatus
```

### 8.4 库存流水流程

```text
新增库存流水
-> 查询库存物品
-> 计算变动前库存
-> 按 changeType 计算变动后库存
-> 校验不能出现负库存
-> 更新 inventory_sku.quantity
-> 写入 inventory_stock_log
```

## 9. 当前运行配置

默认服务端口：

```yaml
server:
  port: 8080
```

默认数据库：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/db_platform
    username: root
    password: 12345678
```

默认 JWT 配置：

```yaml
sky:
  jwt:
    admin-secret-key: itcast
    admin-ttl: 7200000
    admin-token-name: token
```

## 10. 启动和验证

```bash
mvn -pl store-server -am test
```

启动后端：

```bash
mvn -pl store-server -am spring-boot:run
```

默认登录账号：

```text
admin / 123456
staff / 123456
readonly / 123456
```
