# 门店后台管理系统重构接口文档

## 1. 文档说明

本文档用于说明 `beauty-console` 重构后的后台接口、请求参数、返回字段和主要业务规则。

- 接口前缀：`/admin`
- 数据格式：`application/json`
- 字符集：`UTF-8`
- 时间格式：`yyyy-MM-dd HH:mm:ss`
- 日期格式：`yyyy-MM-dd`
- 金额类型：`BigDecimal`
- 主键类型：`Long`
- 分页返回：`Result<PageResult>`
- 普通返回：`Result<T>`

## 2. 统一返回格式

### 2.1 普通返回

```json
{
  "code": 1,
  "msg": null,
  "data": {}
}
```

| 字段 | 类型 | 说明 |
|---|---|---|
| code | Integer | 1 成功，0 失败 |
| msg | String | 返回消息 |
| data | Object | 返回数据 |

### 2.2 分页返回

```json
{
  "code": 1,
  "msg": null,
  "data": {
    "total": 100,
    "records": []
  }
}
```

| 字段 | 类型 | 说明 |
|---|---|---|
| total | Long | 总记录数 |
| records | Array | 当前页数据 |

### 2.3 分页参数

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| page | Integer | 否 | 页码，默认 1 |
| pageSize | Integer | 否 | 每页条数，默认 10，最大 200 |

后端已接入 MyBatis-Plus 物理分页插件，分页接口会按 `page/pageSize` 返回当前页数据。

## 3. 认证说明

除登录接口外，其余后台接口都需要携带 Token。

```http
token: <登录返回的token>
```

当前配置项：

```yaml
sky:
  jwt:
    admin-token-name: token
```

## 5. 用户接口

Controller：`SysUserController`

基础路径：`/admin/users`

Swagger：`@Api(tags = "用户相关接口")`

### 5.1 登录

```http
POST /admin/users/login
```

请求参数：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| username | String | 是 | 登录账号 |
| password | String | 是 | 登录密码 |

请求示例：

```json
{
  "username": "admin",
  "password": "123456"
}
```

本地种子数据提供 6 个测试账号，密码均为 `123456`：

| 账号 | 角色 |
|---|---|
| admin | 超级管理员 |
| manager | 店长 |
| staff | 普通员工 |
| inventory | 库存管理员 |
| finance | 财务/收银 |
| readonly | 只读 |

如果不重置数据库，可执行 `store-server/src/main/resources/sql/patch_test_role_accounts.sql` 补齐这些账号。

返回字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| id | Long | 用户ID |
| username | String | 登录账号 |
| roleId | Integer | 角色ID：1超级管理员，2店长，3普通员工，4库存管理员，5财务/收银，6只读 |
| roleCode | String | 角色编码 |
| roleName | String | 角色名称 |
| staffId | Long | 关联员工ID |
| staffName | String | 员工姓名 |
| status | Integer | 状态：0禁用，1启用 |
| token | String | JWT |

返回示例：

```json
{
  "code": 1,
  "msg": null,
  "data": {
    "id": 1001,
    "username": "admin",
    "roleId": 1,
    "roleCode": "SUPER_ADMIN",
    "roleName": "超级管理员",
    "staffId": 1001,
    "staffName": "测试员工001",
    "status": 1,
    "token": "eyJhbGciOiJIUzI1NiJ9..."
  }
}
```

## 16. 业务规则

### 16.1 登录

- 登录参数必须使用 JSON 请求体。
- 密码字段不返回前端。
- Token 放在统一返回对象 `data.token`。
- 后续请求头名称按配置使用 `token`。

### 16.2 分页

- 分页查询统一返回 `Result<PageResult>`。
- `PageResult.total` 为总记录数。
- `PageResult.records` 为当前页 VO 列表。

### 16.3 保存和修改

- Controller 入参使用 DTO，不直接接收 Entity。
- Controller 出参使用 VO，不直接返回 Entity。
- 新增成功建议返回新增 ID。
- 修改、删除、状态流转建议返回 Boolean。

### 16.4 事务

以下操作必须加事务：

- 新增预约并保存预约项目明细。
- 从预约生成订单。
- 新增订单并保存订单项目明细。
- 收款后同步更新订单支付金额、支付状态、欠款金额。
- 库存变动后同步更新库存数量并写入库存流水。
- 完成服务订单并自动扣减耗材库存。

### 16.5 状态流转

- 预约状态不能随意改数字，应限制为：待确认 -> 已确认 -> 已完成，或取消。
- 已取消预约不能再确认或完成。
- 已取消订单不能再收款或完成。
- 已退款订单不能重复退款。
- 出库数量不能超过当前库存。

### 16.6 库存扣减

- 库存扣减统一通过库存流水服务处理。
- 出库、报损和订单自动扣库存都会校验库存不能小于 0。
- 扣库存时使用 `FOR UPDATE` 锁定库存物品行，避免并发错账。
- 订单自动扣库存只读取启用状态的服务项目耗材关系。

### 16.7 缓存和导出

- 服务项目耗材关系使用 Redis + Spring Cache 缓存。
- 新增、修改、删除服务项目耗材关系时会清理耗材关系缓存。
- 库存物品导出使用 EasyExcel，接口返回 `.xlsx` 文件，不使用统一 JSON 返回格式。
## 本轮新增接口：权限点配置

登录结果新增 `permissions` 字段，前端菜单、工作台入口和按钮会优先按这些权限码显示。

新增接口：

| 接口 | 方法 | 说明 |
|---|---|---|
| `/admin/permissions` | GET | 查询全部权限点 |
| `/admin/users/{id}/permissions` | GET | 查询指定账号当前生效权限 |
| `/admin/users/{id}/permissions` | PUT | 保存指定账号权限点 |

保存权限请求体：

```json
{
  "permissionCodes": ["dashboard:view", "customers:view", "customers:create"],
  "useRoleDefault": false
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| permissionCodes | Array<String> | 是 | 用户级权限编码列表 |
| useRoleDefault | Boolean | 否 | 是否恢复为当前角色默认权限 |

说明：

- `GET /admin/users/{id}/permissions` 返回 `permissionCodes`、`rolePermissionCodes`、`allPermissions` 和 `customized`。
- `rolePermissionCodes` 表示当前账号所属角色的默认权限，供前端“恢复角色默认”按钮回显使用。
- `useRoleDefault=true` 时清空用户级覆盖配置，账号回到当前角色默认权限。
- `useRoleDefault=false` 且 `permissionCodes=[]` 时保存为空的用户级覆盖配置，表示清空该账号权限。
- 非空 `permissionCodes` 会保存为用户级权限覆盖。
