# 执行问题记录

本文单独记录本轮开发、验证和提交过程中遇到的命令、环境、版本管理问题，避免和业务问题混在一起。

## 1. PowerShell 不支持 MySQL `<` 输入重定向

问题：在 PowerShell 里执行：

```powershell
mysql ... db_platform < patch_audit_and_unique_numbers.sql
```

会报错：`The '<' operator is reserved for future use.` 这是 PowerShell 语法限制，不是 SQL 脚本错误。

解决：

```powershell
Get-Content -Raw -Encoding UTF8 store-server\src\main\resources\sql\patch_audit_and_unique_numbers.sql |
  & 'D:\_Dev\Database\MySQL\mysql-8.0.25-winx64\bin\mysql.exe' --default-character-set=utf8mb4 -uroot -p12345678 db_platform
```

执行后已确认：

- `operation_audit_log` 表存在。
- `appointment.uk_appointment_no` 存在。
- `service_order.uk_service_order_no` 存在。
- `payment_record.uk_payment_no` 存在。

## 2. PowerShell 下 Maven `-D` 参数需要加引号

问题：直接执行：

```powershell
mvn -pl store-server -Dstore.real-it=true -Dtest=AdminApiRealIntegrationTest test
```

PowerShell 会把带点的 `-Dstore.real-it=true` 解析异常，Maven 报 `Unknown lifecycle phase ".real-it=true"`。

解决：Windows PowerShell 下把 Maven 属性参数加引号：

```powershell
mvn -pl store-server -am "-Dstore.real-it=true" "-Dtest=AdminApiRealIntegrationTest" "-Dsurefire.failIfNoSpecifiedTests=false" test
```

## 3. 只跑 `store-server` 会丢失多模块依赖

问题：执行真实联调测试时，如果只使用：

```powershell
mvn -pl store-server "-Dstore.real-it=true" "-Dtest=AdminApiRealIntegrationTest" test
```

可能出现 `Type com.wkr.storepojo.entity.ServiceProjectInventory not present`。原因是只选择 `store-server` 模块时，当前 reactor 没有同时构建 `store-pojo` 等依赖模块。

解决：加 `-am`，让 Maven 同时构建被依赖模块：

```powershell
mvn -pl store-server -am "-Dstore.real-it=true" "-Dtest=AdminApiRealIntegrationTest" "-Dsurefire.failIfNoSpecifiedTests=false" test
```

## 4. `-am` 搭配指定测试类会让上游模块报无匹配测试

问题：加 `-am` 后，`store-common`、`store-pojo` 也进入 reactor。它们没有 `AdminApiRealIntegrationTest`，Surefire 会报：

```text
No tests matching pattern "AdminApiRealIntegrationTest" were executed
```

解决：补充：

```text
-Dsurefire.failIfNoSpecifiedTests=false
```

最终可用命令：

```powershell
mvn -pl store-server -am "-Dstore.real-it=true" "-Dtest=AdminApiRealIntegrationTest" "-Dsurefire.failIfNoSpecifiedTests=false" test
```

## 5. PowerShell 不支持当前环境里的 `&&`

问题：提交时直接执行：

```powershell
git add . && git commit -m "feat: ..."
```

在当前 PowerShell 版本里报错：`The token '&&' is not a valid statement separator`。

解决：拆成两条命令执行：

```powershell
git add .
git commit -m "feat: ..."
```

## 6. `rg` 搜索以 `-D` 开头的文本需要 `--`

问题：使用 `rg` 搜索 `-Dstore.real-it` 时，如果不加 `--`，`rg` 会把 `-D...` 当作命令参数解析。

解决：

```powershell
rg -n -g "*.md" -- "-Dstore\.real-it"
```
